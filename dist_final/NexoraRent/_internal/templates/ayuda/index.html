📘 Centro de Ayuda - ERP Gestión de Alquileres

Este sistema permite gestionar propiedades tanto en alquiler vacacional como en larga duración, incluyendo propiedades, reservas, contratos, ingresos, gastos, tareas e informes de rentabilidad.

👤 Usuarios y Licencias

  • Al registrarte, accedes con un plan DEMO con limitaciones iniciales.
  • El sistema controla automáticamente:
    • Número máximo de propiedades
    • Número máximo de reservas
    • Fecha de caducidad de la licencia
  • Tipos de licencia disponibles:
    • DEMO: limitada
    • TRIAL: completa por tiempo limitado
    • ACTIVA: completa y renovable
    • VITALICIA: completa sin caducidad
  • Si la licencia caduca, el sistema entra en modo restringido:
    • Puedes acceder y consultar datos
    • No puedes realizar nuevas operaciones
    • No se pierde ninguna información

🏠 Propiedades

  • Una propiedad es la base principal del sistema.
  • Puedes gestionar:
    • Viviendas completas
    • Propiedades por habitaciones
  • Cada propiedad puede incluir:
    • Habitaciones
    • Precios
    • Datos fiscales
    • Datos técnicos y descriptivos

🛏️ Habitaciones

  • Permiten trabajar tanto con alquiler completo como con alquiler por habitaciones.
  • El sistema controla automáticamente la disponibilidad y la ocupación.

📅 Reservas (Vacacional)

  • Las reservas representan alquileres de corta duración.
  • Incluyen:
    • Fechas de entrada y salida
    • Huéspedes
    • Asignación de habitaciones
    • Origen de la reserva (Booking, Airbnb, directo, etc.)
  • El sistema puede calcular automáticamente:
    • Precio total
    • Depósito
    • Saldo pendiente
  • Si una propiedad tiene un contrato activo de larga duración en fechas compatibles, no podrá utilizarse para reservas vacacionales.

👥 Huéspedes

  • Cada reserva puede tener uno o varios huéspedes.
  • Se pueden preparar datos para documentos oficiales como:
    • Registro de viajeros

📄 Contratos (Larga Duración)

  • Se utilizan para alquileres de media o larga duración.
  • Incluyen:
    • Arrendatarios o inquilinos
    • Fechas de contrato
    • Importe mensual
    • Condiciones principales
  • Si la propiedad tiene reservas vacacionales en fechas que entren en conflicto, no se permitirá crear el contrato hasta que esas fechas queden libres.

🧾 Recibos

  • Se generan a partir de contratos de larga duración.
  • Representan:
    • Renta mensual
    • Pagos del inquilino
    • Seguimiento de cobros

💡 Suministros (Larga Duración)

  • Se registran mediante lecturas de contador cuando corresponda.
  • Funcionamiento habitual:
    • Se registra lectura
    • Se calcula consumo
    • Se repercute al inquilino
  • Importante:
    • No se tratan como gasto directo de explotación si se repercuten al inquilino
    • Se consideran importes facturados o repercutidos

💰 Ingresos

  • Incluyen:
    • Pagos de reservas vacacionales
    • Pagos de recibos de alquiler
    • Otros ingresos vinculados a la propiedad

💸 Gastos

  • Se registran por propiedad para medir correctamente la rentabilidad.
  • Ejemplos:
    • Limpieza
    • Mantenimiento
    • Reparaciones
    • Facturas reales de suministros

📊 Rentabilidad

  • El sistema distingue dos modelos de explotación:
🏖️ Vacacional
  • Ingresos por reserva
  • Gastos de la vivienda repartidos
  • Comisiones de plataformas incluidas
🏠 Larga duración
  • Ingresos por rentas
  • Suministros repercutidos al inquilino
  • Gastos reales asociados a la vivienda
La rentabilidad se calcula de forma independiente según el tipo de explotación, para no mezclar vacacional y larga duración de forma incorrecta.

📧 Configuración de correo

  • Cada cliente puede configurar su propio correo SMTP desde su perfil.
  • Esto permite enviar comunicaciones sin depender del correo del desarrollador.
  • Se recomienda revisar la configuración antes de usar avisos o envíos automáticos.

🤖 Simulador

  • Permite generar datos de prueba y simulaciones para demostraciones.
  • Se utiliza para:
    • Probar el sistema
    • Detectar errores
    • Simular funcionamiento comercial
  • También puede limpiar datos de prueba generados.
  • En entornos de producción puede estar oculto o desactivado.

🌐 Acceso al sistema

  • El sistema puede utilizarse de diferentes formas:
    • 💻 Instalación local en Windows, Linux o Mac
    • 🌍 Acceso web mediante servidor para demostración o uso remoto
    • 📱 Acceso desde móvil o tablet mediante navegador
  • La gestión completa se recomienda desde ordenador.
  • En móviles y tablets el uso recomendado es consulta, seguimiento y gestión ligera.

🎯 Objetivo del sistema

  • Control total de la explotación inmobiliaria
  • Diferenciar correctamente entre vacacional y larga duración
  • Optimizar la rentabilidad de cada propiedad
  • Preparar futuras mejoras e integraciones
